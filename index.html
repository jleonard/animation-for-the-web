Introduction

There are many techniques for creating animations for the web. We're going to focus on 2 methods for making css based animations. css animations are particularly useful when you want to animate user interfaces (as opposed to making games, animated data visualizations etc.).

recap

You know what .css and .html are don't you? The quickest way to explain is that html is structure and css is style.

Here's some sample html. It's structred content.

<h1>I'm a headline</h1>
<p>I'm a paragraph. Lorem ipsum dolor...</p>

...and here are css rules that apply style to the structure.

h1{
  font-size: 80px;
  font-family: Arial;
  color: rgb(20,20,20);
}
p{
  font-size: 12px;
  font-family: Georgia;
}

...and if that's too complicated

.html .html + .css

transitions

Transitions allow you to change one or more style properties over time.

transition
transition

Both boxes have hover effects that change the text and background colors. The box on the left has a 2 second linear transition applied while the right showcases the standard hard swap behavior.

Transitions are made up of 4 different variables

section{
  transition-property: background-color;
  transition-duration: 2s;
  transition-delay: 0s;
  transition-timing-function: ease-in;
}
property the style properties to animate (ex: background-color, width)
duration expressed in seconds or milliseconds
delay optional a delayed start time expressed in seconds or milliseconds
timing function the style of the transition expressed as an acceleration curve (more on this below).

timing transitions

.css has a number of built-in timing functions to control a transition's acceleration.

These timing functions are based on cubic bezier curves.

Notice they don't have any effect on the duration.
