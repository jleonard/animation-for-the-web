Introduction

There are many techniques for creating animations for the web. We're going to focus on 2 methods for making css based animations. css animations are particularly useful when you want to animate user interfaces (as opposed to making games, animated data visualizations etc.).

transitions

.css transitions change one or more style properties over time.

transition
transition

Both boxes have hover effects that change the text and background colors. The box on the left has a 2 second linear transition while the right shows the standard hard swap behavior.

Transitions are defined by 4 properties.

property the style properties to animate (ex: background-color, width).
duration length of time the transition takes to get from value A to value B.
acceleration (aka: timing function) the transition's acceleration (more on this below).
delay *optional* length of time to wait before starting a transition.

Code Example

section{
  transition-property: background-color;
  transition-duration: 2s;
  transition-delay: 0s;
  transition-timing-function: ease-in;
}

understanding acceleration

In .css acceleration is referred to as a timing-function. The terms acceleration and timing-fuctions are used interchangably in this section.

Acceleration sounds similar to duration but each has a unique responsibility.

duration is the fixed length of time it takes to complete a transition.

acceleration is the change in velocity that occurs during the transition. Example: you may want a transition that starts quickly and slows to a crawl at the end.

.css has a number of built-in timing functions. Test them below and note how each has a different acceleration while they share the same duration.

The timing-functions are based on cubic bezier curves. When represented visually they should look familiar to anyone that's used line drawing tools like Adobe Illustrator.

The curves are defined by two main points (start and stop) and two control points.

creating your own acceleration styles

The out-of-the-box timing functions have been around for 1,000,000 years. They're not very emotive and they lack physicality.

If you want finer control to create a more expressive animation then you'll need to create your own custom bezier curves. The easiest way is to use an online bezier builder.

keyframe animations

how are animations and transitions different?

the biggest contrast with transitions is that animations give you control over the steps or keyframes

In this example there are three keyframes that define the state of the object at 0%, 80% and 100% of the animation.

keyframes
@keyframes slide {
  0% {
    transform: translateX(0%);
  }
  80% {
    background-color: transparent;
    transform: translateX(-10%) skew(-5deg, -5deg);
  }
  100% {
    background-color: #ffdc00;
    color: #ff4136;
    transform: translateX(200%) skew(5deg, 5deg);
  }
}
