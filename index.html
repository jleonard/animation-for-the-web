Introduction

There are many techniques for creating animations for the web. We're going to focus on 2 methods for making css based animations. css animations are particularly useful when you want to animate user interfaces (as opposed to making games, animated data visualizations etc.).

Recap

You know what .css and .html are don't you? The quickest way to explain is that html is structure and css is style.

Here's some sample html. It's structred content.

<h1>I'm a headline</h1>
<p>I'm a paragraph. Lorem ipsum dolor...</p>

...and here are css rules that apply style to the structure.

h1{
  font-size: 80px;
  font-family: Arial;
  color: rgb(20,20,20);
}
p{
  font-size: 12px;
  font-family: Georgia;
}

...and if that's too complicated

.html .html + .css

Get it? Let's animate.

.css transitions

Transitions allow you to change one or more style properties over time.

transition
transition

In the example above both boxes are styled to have a solid background with white text on hover. The first box has a transition rule that says anytime a change in color occurs, a 2 second transition will move between the values. The second box has no transition applied so the color changes happen instantaneously.

.demo{
  transition-duration: 2s;
  transition-property: all;
  color: #222;
}
.demo:hover{
  background-color: #ffdc00;
  color: #fff;
}

Timing transitions

.css has a number of built-in timing functions to control a transition's acceleration.

Test

Or you can create your own custom acceleration function. The timing functions are cubic bezier curves just like Illustrator's pen tool. The curves are based on a start and end point with two control points.
Here's what a custom equation looks like... 

cubic-bezier(1,.1,1,.5)

Getting a transition right is about balancing the acceleration and the duration. Go play with this transition visualizer.
